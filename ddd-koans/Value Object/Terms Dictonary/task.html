Domain terms

  • Time signature - Symbols that describe repeating beat rhythm in a piece of music. It consists of the number of beats and the note value.
  • Number of beats - The upper number of time signature indicates how many beats constitute a bar.
  • Note value - The lower number of time signature always corresponds to the beat unit which is a value of a music note.
  • Bar - A written up piece of music is divided into chunks called bars. Each bar follows a time signature assigned at the beginning of the bar until a new time signature occurs.
  • Note - Symbol is denoting a musical sound. Notes can represent the pitch and duration of a sound in musical notation.

Domain-driven design

  • Value object - domain-driven design building block, an immutable type that is defined by its state properties. The value object should be also self-validating.
  • Immutability - property of the object or data structure that prevents its state changes after its creation
  • Property-based equality - equality based only on the values of the object without using any reference to the object's identity.
  • Tiny type - Type typically with one property, that is just wrapper around other non-domain value. Representing such non-domain value as a simple value object prevents lots of errors and ensures that all of the objects has domain meaning. Example class: NumberOfBeats, NoteValue.
