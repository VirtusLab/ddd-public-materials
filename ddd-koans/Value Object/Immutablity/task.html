In this task and at least for the next few you will develop a TimeSignature class.
It could serve as a great example of a value object.
So here we go with a reminder of what time signatures are:

Time signatures are symbols which describe repeating beat rhythm in a piece of music.
It tells us the total length of all symbols in a bar.
The upper value is called a numerator and the lower value is a denominator.

As the description states TimeSignature must contain only two properties:
  • numerator,
  • denominator,
Both properties should be of type Int.

Immutability is one of the characteristics of a Value Object.
It means that once a value object is created, it can never be changed. So the state is unchangeable.


Implement the TimeSignature class in a way that will ensure immutability.

Both properties should be declared using val not var .
Remember not to use nullable type Int?.
