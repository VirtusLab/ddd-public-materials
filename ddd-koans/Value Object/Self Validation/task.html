TimeSignature is described using two properties, the same you have declared in the previous task.
However, values of these properties cannot be random integers. There are domain rules. If you don't remember, here you go:

  • number of beats is a number between 1 and 32,
  • note value is a number between 1 and 32,
  • note value is a power of two.

How should we validate the values? Let your code throw an exception of any kind on invalid values.
Consider using require function.

To check if the number is a power of two we provided extension function Int.main.isPowerOfTwo().

This may not be obvious at first glance, but if you correctly check if note value is in a power of two, then checking if note value is between 1 and 63 would make the test pass as values between 32 and 63 are covered by the power of two conditions.
