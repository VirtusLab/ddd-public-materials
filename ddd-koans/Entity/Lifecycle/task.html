Lifecycle

In the beginning, while creating a piece of music, bars don't exist in the piece at all. Then creators add bars one after another. They decide what time signature should be used and what notes should fill the bar.

As the definition of the bar states, there is a limit on notes in the bar. So creators can add notes just to fill the limit. But, the bar is only complete when is filled with enough notes to match the beats corresponding to the bar's time signature.

To recall information about time signature in a bar:

  • The note value of the time signature defines which note value is equal to one beat.
  • The number of beats declares how many beats makes a complete bar.
Therefore, to create a complete bar, the sum of beats of note values in the bar must be equal to the time signature's number of beats.

To sum up, a bar can not yet exist, be in an incomplete state or be in a complete state. The lifecycle must be modelled in the code then.

Task

To begin with the lifecycle, please implement Bar#state() method which returns Bar.State.Complete for a complete bar, Bar.State.Incomplete for an incomplete bar and throws BarWithTooManyNotesException when there are too many notes in the bar.
